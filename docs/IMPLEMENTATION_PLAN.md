# MouseIncMac 分阶段实现计划

最后更新：2026-07-11

本文档是 MouseIncMac 的实施路线图，用于组织开发顺序、并行边界、验收条件和发布门槛。功能映射与技术边界见 [PORTING_SPEC.md](PORTING_SPEC.md)。

## 1. 总体目标

在不复用 Windows 二进制的前提下，以 macOS 原生能力重建 MouseInc 的核心交互，并满足以下恒定原则：

- 普通右键和 macOS 原生触控板行为不得因功能扩展而回归。
- 默认离线，不上传按键、剪贴板、屏幕内容或 OCR 数据。
- 权限按需申请；新功能不得无条件扩大 P0/P1 的权限集。
- 所有共享数据结构都带 schema 版本、迁移逻辑和回归测试。
- 每个阶段只在自动化验证和真机验收同时通过后关闭。

## 3. P0：工程与核心基线

状态：已完成。

### 3.1 已交付

- Swift Package 分离 `MouseIncCore` 与 `MouseIncMac`。
- `kCGHIDEventTap` 右键 Down/Dragged/Up 输入链路。
- 基于 HID delta 累计的 `GestureSession` 状态机。
- 普通右键在 `kCGSessionEventTap` 层回放，避免重新进入 HID 钩子。
- 上、下、左、右和折线识别，默认五个动作。
- `schemaVersion: 2` 配置，旧扁平 JSON 自动备份与迁移。
- XCTest 与 `MouseIncCoreCheck` 双重回归入口。
- Release 构建、Apple Development 签名、安装和重启脚本。

### 3.2 回归门槛

- `swift test`
- `swift run MouseIncCoreCheck`
- `swift build -c release`
- `codesign --verify --deep --strict ~/Applications/MouseIncMac.app`
- 普通右键、成功手势、未绑定手势和超时手势的真机验收

## 4. P1：触控板兼容

状态：已完成并经用户真机验收。

### 4.1 已交付

- 不判断设备来源；鼠标和触控板的右键事件进入同一状态机。
- 触控板使用“辅助点击并按住 → 拖动 → 松开”绘制轨迹。
- 短点击回放普通右键；跨过阈值后超时则取消手势，不弹出菜单。
- 轨迹显示、最大持续时间和失败提示可通过 JSON 设置。
- 菜单栏展示触控板用法并可打开系统触控板设置。
- 双指滚动、捏合、Mission Control 和系统三/四指手势完全交给 macOS。

### 4.2 不可回退的边界

- 不引入私有 `MultitouchSupport`、IOKit SPI 或虚拟 HID 驱动。
- 不将输入监控权限作为 P0/P1 启动条件。
- 不用 `.cgSessionEventTap` 替代 HID 层的右键捕获。
- 不依赖被拦截后的全局光标位置生成轨迹。

## 5. P2-0：双轨共享契约

状态：已完成。

目标：先锁定 P2A/P2B 都需要的数据与执行接口，之后两轨才正式并行。

### 5.1 实现顺序

1. [x] 定义动作序列语义：顺序执行、延时、取消和错误策略。
2. [x] 将 `ActionDefinition.Kind` 演进为可扩展的动作目录，保留旧 `keyStroke/openURL/launchApplication` 兼容。
3. [x] 设计 schema 3，完成 schema 2 → 3 迁移、备份和未来版本拒绝策略。
4. [x] 抽象权限状态：辅助功能、屏幕录制、输入监控各自独立。
5. [x] 定义本地诊断事件格式，不记录按键内容、剪贴板内容或截图像素。
6. [x] 合并契约与迁移测试，为 P2A-1 与 P2B-1 解锁并行开发。

### 5.2 建议模块边界

```text
Sources/MouseIncCore/
  Configuration/      # schema、迁移、规则匹配
  Gestures/           # 轨迹、识别、模板
  Actions/            # 动作定义、序列、验证

Sources/MouseIncMac/
  Input/              # CGEventTap、触发器、输入状态
  ActionExecution/    # macOS 动作执行器
  Windowing/          # Accessibility 窗口操作
  Settings/           # SwiftUI 设置
  Capture/            # ScreenCaptureKit
  OCR/                # Vision
  Presentation/       # 轨迹、贴图、画中画
```

### 5.3 验收

- P0/P1 的 schema 2 配置可无损升级。
- 一个手势可顺序执行多个动作并在指定节点延时。
- 不同权限状态不会阻止无关功能启动。
- P2A/P2B 新文件不需要同时改动手势钩子与截图引擎。

## 6. P2A：输入与效率轨

### P2A-1：手势与动作引擎

状态：进行中。八方向量化、定向归一化模板识别、圆形和 C/M/Z 字母样本已完成。经 2026-07-12 验收决定，字母容错优化与自定义录制/训练 API 暂挂，不作为当前发布门槛；后续重启时采用用户多样本、分类阈值和候选分差方案。经 2026-07-13 决定，触控板断点续画也暂挂，详见下方任务定义。完整按键生命周期仍待实现。窗口居中作为 P2A-2 的首个纵向切片已接入动作目录。

交付：

- 八方向量化和对角线识别。
- 圆形、字母和自定义模板的归一化与相似度匹配。
- 手势录制、重命名、训练和冲突检查的核心 API。
- 在 P2-0 动作序列上扩展快捷键按下/抬起和后续新动作。

验收：

- 现有五个手势结果不变。
- 对角线、圆形和至少三个字母模板在固定样本集上达到可重复的识别结果。
- 动作序列中的延时可取消，应用退出时不留下按住的合成按键。

#### 暂挂：触控板断点续画

目的：解决触控板面积有限、复杂或大范围手势无法一次画完的问题。用户在已经进入手势状态后可以短暂抬起手指；应用不立即识别，而是在一个可配置的宽限期内保留轨迹。用户再次进行辅助按下并拖动时，从原轨迹末端继续绘制；宽限期结束且没有继续输入时才识别和执行。

约束与验收边界：

- 不判断事件来自鼠标还是触控板，仍复用 P1 的统一右键事件链路；因此鼠标快速松开再按下也遵循同一规则。
- 只有已经越过启动距离、进入手势状态的轨迹可以等待续画；普通短右键必须继续及时回放，不能产生明显延迟。
- 等待期间不得弹出上下文菜单、提前执行动作或丢失现有轨迹；续画后的轨迹、包围框和截图范围必须作为一个手势计算。
- 需要定义宽限期、再次按下的空间容差、明确取消方式，以及应用切换、超时、睡眠和权限变化时的清理策略。
- 默认值和是否默认开启须经真机手感测试后决定；该功能当前不作为发布门槛。

### P2A-2：窗口与指针增强

状态：进行中。已交付居中、macOS 独立空间全屏切换、最小化和关闭窗口；全屏动作直接发送系统 `Control + Command + F`，同一手势再次触发即退出全屏。窗口置顶和 Option 拖窗并非已确认的原版复刻范围，移出近期实现计划；只有在原版功能完成后、获得单独确认时才作为扩展能力评估。边缘滚轮、热角和音量仍以原版行为和拆解结果为准，后续排期。

交付：

- Accessibility 窗口选择、最大化/还原、最小化、居中和关闭能力。
- 屏幕边缘滚轮、热角和按应用排除。
- 音量、静音和应用启动动作。

验收：

- Finder、Safari、Electron 应用和至少一个多窗口应用通过窗口动作矩阵。
- 多屏边缘、共享边界和不同排列方式不会重复触发。

### P2A-3：剪贴板与旧配置迁移

交付：

- 连续复制触发的剪贴板菜单。
- 搜索、翻译、URL/Base64 编解码、哈希和二维码动作。
- 快速粘贴与可选的选中文字自动复制。
- Windows `MouseInc.json` 的只读导入、映射预览和不支持项报告。

验收：

- 默认不持久化剪贴板历史，日志不包含剪贴板内容。
- 密码框、安全输入和被排除应用不触发自动复制/菜单。
- 导入旧配置只生成预览，未经用户确认不执行 URL、程序或脚本。

交付：

- 所有置顶窗口都能独立关闭，应用退出后不残留捕获会话。

## 8. 双轨并行规则

### 8.1 文件所有权

| 范围 | 主责轨道 | 修改规则 |
|---|---|---|
| `MouseIncCore/Configuration` | 共享 | 先合并设计与迁移测试，A/B 不得各自扩展 schema |
| `MouseIncCore/Gestures` | P2A | B 轨只调用录制/预览 API，不修改识别逻辑 |
| `MouseIncCore/Actions` | 共享 | A 定义执行语义，B 使用同一契约构建编辑器 |
| `MouseIncMac/Input` | P2A | B 不得为设置或截图改动 HID 事件 tap |
| `MouseIncMac/Windowing` | P2A | 通过动作接口暴露，不直接依赖设置 UI |
| `MouseIncMac/Settings` | P2B | 通过配置存储 API 读写，不直接操作输入状态 |
| `MouseIncMac/Capture` / `OCR` | P2B | 屏幕权限失败不得影响 A 轨功能 |

### 8.2 合流规则

- 共享模型改动先独立合流，随后 A/B 各自升级调用方。
- 每个功能绑定独立权限；不把屏幕录制或输入监控变成全局前置条件。
- 所有可执行动作必须先支持配置校验和可读的 UI 描述。
- 新输入监听不得在主事件 tap 回调中执行文件、网络或 Accessibility 重操作。
- 冲突时以 P0/P1 的右键可用性和原生触控板行为为最高优先级。

## 9. 通用完成标准

每个子阶段必须同时满足以下条件：

1. 功能路径有自动测试，错误和取消路径不得只靠手工验证。
2. `swift test`、`MouseIncCoreCheck` 和 Release 构建通过。
3. 安装后签名验证通过，Bundle ID 保持 `com.mason.mouseincmac`。
4. 旧配置可迁移，迁移前备份，新配置可往返编解码。
5. 权限拒绝、运行中撤销和权限恢复后的行为明确。
6. 手动验收覆盖鼠标、内建触控板或 Magic Trackpad，以及相关的多屏场景。
7. 诊断日志能证明状态转移，但不包含敏感用户内容。
8. README、移植规格和本实现计划与当前行为一致。

## 10. 下一实施批次

共享前置、八方向手势、基础窗口动作与设置编辑闭环均已完成。后续以 Windows 原版已确认能力为唯一近期优先级，不将通用窗口增强工具功能混入复刻路线。

### 当前批：P2B-2 最小闭环（原版方框手势）

“方框手势 → 直接截取手势包围范围 → 生成贴图”候选版本已实现，当前进入真机验收；方框模板容错优化仍按决定暂挂，可暂用同时具有横纵跨度的其他已绑定手势验证截图范围。通过后，以“反向方框手势 → Vision OCR → 结果预览与按需复制”接入离线 OCR；实现反向方框前必须先让模板识别器保留绘制方向，不能复用当前正反路径等价策略。

验收重点：拒绝屏幕录制权限时 P0/P1 与窗口动作完全不受影响；单屏及扩展屏的框选坐标正确；贴图不泄露捕获内容到网络。

### 并行收尾：P2B-1

设置页继续补齐拖放排序和完整分页导航，以便为截图、贴图与 OCR 动作提供可配置入口。

### 后续队列

完成 P2B-2 后，按原版能力继续评估边缘滚轮、热角、音量和剪贴板功能；窗口置顶与 Option 拖窗不在复刻发布门槛内，除非另行确认。
