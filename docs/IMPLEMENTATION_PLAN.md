# MouseTrails 分阶段实现计划

最后更新：2026-07-14

本文档是 MouseTrails 的实施路线图，用于组织开发顺序、并行边界、验收条件和发布门槛。功能映射与技术边界见 [PORTING_SPEC.md](PORTING_SPEC.md)。

## 1. 总体目标

在不复用 Windows 二进制的前提下，以 macOS 原生能力重建 MouseInc 的核心交互，并满足以下恒定原则：

- 普通右键和 macOS 原生触控板行为不得因功能扩展而回归。
- 默认离线，不上传按键、剪贴板、屏幕内容或 OCR 数据。
- 权限按需申请；新功能不得无条件扩大 P0/P1 的权限集。
- 所有共享数据结构都带 schema 版本、迁移逻辑和回归测试。
- 每个阶段只在自动化验证和真机验收同时通过后关闭。

## 3. P0：工程与核心基线

状态：已完成。

### 3.1 已交付

- Swift Package 分离 `MouseIncCore` 与 `MouseIncMac`。
- `kCGHIDEventTap` 右键 Down/Dragged/Up 输入链路。
- 基于 HID delta 累计的 `GestureSession` 状态机。
- 普通右键在 `kCGSessionEventTap` 层回放，避免重新进入 HID 钩子。
- 上、下、左、右和折线识别，默认五个动作。
- `schemaVersion: 2` 配置，旧扁平 JSON 自动备份与迁移。
- XCTest 与 `MouseIncCoreCheck` 双重回归入口。
- Release 构建、Apple Development 签名、安装和重启脚本。

### 3.2 回归门槛

- `swift test`
- `swift run MouseIncCoreCheck`
- `swift build -c release`
- `codesign --verify --deep --strict ~/Applications/MouseTrails.app`
- 普通右键、成功手势、未绑定手势和超时手势的真机验收

## 4. P1：触控板兼容

状态：已完成并经用户真机验收。

### 4.1 已交付

- 不判断设备来源；鼠标和触控板的右键事件进入同一状态机。
- 触控板使用“辅助点击并按住 → 拖动 → 松开”绘制轨迹。
- 短点击回放普通右键；跨过阈值后超时则取消手势，不弹出菜单。
- 轨迹显示、最大持续时间和失败提示可通过 JSON 设置。
- 菜单栏展示触控板用法并可打开系统触控板设置。
- 双指滚动、捏合、Mission Control 和系统三/四指手势完全交给 macOS。

### 4.2 不可回退的边界

- 不引入私有 `MultitouchSupport`、IOKit SPI 或虚拟 HID 驱动。
- P0/P1 只以辅助功能权限作为启动条件。
- 不用 `.cgSessionEventTap` 替代 HID 层的右键捕获。
- 不依赖被拦截后的全局光标位置生成轨迹。

## 5. P2-0：双轨共享契约

状态：已完成。

目标：先锁定 P2A/P2B 都需要的数据与执行接口，之后两轨才正式并行。

### 5.1 实现顺序

1. [x] 定义动作序列语义：顺序执行、延时、取消和错误策略。
2. [x] 将 `ActionDefinition.Kind` 演进为可扩展的动作目录，保留旧 `keyStroke/openURL/launchApplication` 兼容。
3. [x] 设计 schema 3，完成 schema 2 → 3 迁移、备份和未来版本拒绝策略。
4. [x] 抽象权限状态：辅助功能与屏幕录制各自独立。
5. [x] 定义本地诊断事件格式，不记录按键内容、剪贴板内容或截图像素。
6. [x] 合并契约与迁移测试，为 P2A-1 与 P2B-1 解锁并行开发。

### 5.2 建议模块边界

```text
Sources/MouseIncCore/
  Configuration/      # schema、迁移、规则匹配
  Gestures/           # 轨迹、识别、模板
  Actions/            # 动作定义、序列、验证

Sources/MouseIncMac/
  Input/              # CGEventTap、触发器、输入状态
  ActionExecution/    # macOS 动作执行器
  Windowing/          # Accessibility 窗口操作
  Settings/           # SwiftUI 设置
  Capture/            # ScreenCaptureKit
  OCR/                # Vision
  Presentation/       # 轨迹、贴图与视觉反馈
```

### 5.3 验收

- P0/P1 的 schema 2 配置可无损升级。
- 一个手势可顺序执行多个动作并在指定节点延时。
- 不同权限状态不会阻止无关功能启动。
- P2A/P2B 新文件不需要同时改动手势钩子与截图引擎。

## 6. P2A：输入与效率轨

### P2A-1：手势与动作引擎

状态：已完成。八方向量化、定向归一化 W 模板识别、顺逆时针方框、搜索选中文字和可切换轨迹颜色均已完成；轨迹颜色通过 macOS 原生调色板选择并保存 sRGB RGBA 值。旧的内置 S 模板已移除，“搜索选中文字”改为在交互教程中引导用户录制三次自定义轨迹（推荐 S），训练成功后写入真实配置与教程临时配置。自定义手势采用 64 点归一化、最佳模板分数、最低阈值和前两候选分差方案；训练阶段会拒绝过短或不一致样本并提示冲突。W 字母容错优化继续暂挂。

交付：

- 八方向量化和对角线识别。
- W、顺逆时针方框和自定义模板的归一化与相似度匹配。
- 手势三次录制、训练、预览、重新录制和冲突检查的完整闭环。
- 在 P2-0 动作序列上扩展快捷键按下/抬起和后续新动作。

验收：

- 现有五个手势结果不变。
- 对角线、W、顺逆时针方框和用户自定义模板在固定样本集上达到可重复的识别结果。
- 动作序列中的延时可取消，应用退出时不留下按住的合成按键。

### P2A-2：窗口与指针增强

状态：已完成。已交付居中、macOS 独立空间全屏切换、最小化和关闭窗口；全屏动作直接发送系统 `Control + Command + F`，同一手势再次触发即退出全屏。边缘滚轮已验收：左边亮度、右边音量，触控板连续滚动已降速。窗口置顶、Option 拖窗、热角、静音及额外应用启动能力均已移出范围。

交付：

- Accessibility 窗口选择、最大化/还原、最小化、居中和关闭能力。
- 屏幕左边缘滚动调节亮度、右边缘滚动调节音量。
- 静音、额外应用启动、热角、窗口置顶和 Option 拖窗不在当前范围。

验收：

- Finder、Safari、Electron 应用和至少一个多窗口应用通过窗口动作矩阵。
- 多屏左/右边缘、共享边界和不同排列方式不会重复调节；非边缘滚动不受影响。

### P2A-3：剪贴板与旧配置迁移

状态：已移出范围。快速粘贴与手势绑定 `Command+V` 的价值重叠；剪贴板历史、文本工具和旧版 JSON 导入均不进入当前产品计划。

## 7. P2B：视觉与设置轨

### P2B-1：SwiftUI 设置基础

状态：已完成。设置窗口已重构为侧边栏分区架构，依次呈现通用、手势绑定、边缘滚动、权限、配置；普通设置页采用统一居中的响应式内容轴，缩放后仍保持标题、表单和保存栏对齐。原“贴图与 OCR”说明页已移除，相关内容迁入首次启动的任务式交互教程。教程只展示当前任务，使用内存中的临时默认配置执行真实复制粘贴、浏览、搜索、隔离窗口、贴图和 OCR 效果；错误手势被安全拦截，成功结果经验证后以 👍 与短时粒子动画反馈。贴图教学依次检测拖动、折叠、另存为、恢复、透明度、复制和用户右键关闭，每项完成后显示勾选状态；系统为屏幕录制权限重启应用时不会误记教程完成。手势绑定采用响应式主从布局，窗口较宽时左右并排，缩窄时将列表折叠为带缩略图的顶部选择栏，让编辑器保留完整宽度；轨迹缩略图带末端方向箭头，默认“新手势”名称会根据首个动作自动更新，并保留草稿编辑、实时校验、原子保存、上下排序、应用选择器、配置导出和备份恢复。权限页的未授权项现支持定向跳转系统设置，并弹出置顶的可拖拽应用授权助手，授权过程会自动刷新设置页和菜单栏状态。内置配置只在全新安装时生成，不覆盖升级用户数据；退出应用动作保留为可选动作，但不再默认绑定。

交付：

- 通用、手势、动作、应用规则、权限与诊断页面，以及首次启动/通用页可回看的交互教程。
- 配置编辑使用草稿副本，保存时统一校验并原子写入。
- 手势轨迹预览、绑定排序、应用选择和冲突提示。
- 辅助功能与屏幕录制分开展示，并在首次教程中解释各自用途。

验收：

- 所有默认配置都能在 UI 中完整往返，不丢字段。
- 非法快捷键、未知动作和冲突绑定在保存前就能解释错误。
- 用户可导出、备份、恢复配置，且不改变应用签名身份。

### P2B-2：截图、贴图与 OCR

状态：已完成最终真机复验。截图、基础贴图、Vision 离线中英文 OCR、自动复制与截断摘要通知已通过验收；截图管线保留完整屏幕合成内容，可识别 MouseTrails 自身的设置窗口，不再穿透到后方应用，并在截图时单独排除手势轨迹覆盖层。贴图支持 20%–100% 透明度调节、多贴图独立生命周期、展开态 Command+C 复制，以及缩小态右键另存 PNG；贴图 OCR、方向键移动和 Option 修饰键均已移除以保持交互简洁。方框容错采用三条相邻边访问与宽松趋势评分，区分 `SQUARE_CLOCKWISE` / `SQUARE_COUNTERCLOCKWISE`，已通过真机验收。

交付：

- ScreenCaptureKit 显示器列表、跨屏区域原生倍率捕获和屏幕录制权限协调。
- 手势范围截图、复制、PNG 保存和透明背景处理。
- 无边框置顶贴图，支持拖动、缩放、折叠、悬停滚动调节透明度、展开态复制、缩小态另存和多窗口。
- Vision 离线 OCR，支持语言识别、自动复制和截断摘要通知。

验收：

- 单屏、左/右/上方扩展屏、负坐标和不同 Retina 缩放组合通过样本矩阵。
- 拒绝屏幕录制权限时，鼠标/触控板手势与窗口动作仍可使用。
- OCR 不发起网络请求，结果只在用户主动触发 OCR 动作时写入剪贴板。

## 8. 双轨并行规则

### 8.1 文件所有权

| 范围 | 主责轨道 | 修改规则 |
|---|---|---|
| `MouseIncCore/Configuration` | 共享 | 先合并设计与迁移测试，A/B 不得各自扩展 schema |
| `MouseIncCore/Gestures` | P2A | B 轨只调用录制/预览 API，不修改识别逻辑 |
| `MouseIncCore/Actions` | 共享 | A 定义执行语义，B 使用同一契约构建编辑器 |
| `MouseIncMac/Input` | P2A | B 不得为设置或截图改动 HID 事件 tap |
| `MouseIncMac/Windowing` | P2A | 通过动作接口暴露，不直接依赖设置 UI |
| `MouseIncMac/Settings` | P2B | 通过配置存储 API 读写，不直接操作输入状态 |
| `MouseIncMac/Capture` / `OCR` | P2B | 屏幕权限失败不得影响 A 轨功能 |

### 8.2 合流规则

- 共享模型改动先独立合流，随后 A/B 各自升级调用方。
- 每个功能绑定独立权限；不把屏幕录制变成手势监听的全局前置条件。
- 所有可执行动作必须先支持配置校验和可读的 UI 描述。
- 新输入监听不得在主事件 tap 回调中执行文件、网络或 Accessibility 重操作。
- 冲突时以 P0/P1 的右键可用性和原生触控板行为为最高优先级。

## 9. 通用完成标准

每个子阶段必须同时满足以下条件：

1. 功能路径有自动测试，错误和取消路径不得只靠手工验证。
2. `swift test`、`MouseIncCoreCheck` 和 Release 构建通过。
3. 安装后签名验证通过，Bundle ID 保持 `com.mason.mouseincmac`。
4. 旧配置可迁移，迁移前备份，新配置可往返编解码。
5. 权限拒绝、运行中撤销和权限恢复后的行为明确。
6. 手动验收覆盖鼠标、内建触控板或 Magic Trackpad，以及相关的多屏场景。
7. 诊断日志能证明状态转移，但不包含敏感用户内容。
8. README、移植规格和本实现计划与当前行为一致。

## 10. 下一实施批次

共享前置、八方向手势、基础窗口动作与设置编辑闭环均已完成。后续以 Windows 原版已确认能力为唯一近期优先级，不将通用窗口增强工具功能混入复刻路线。

### 已完成验收：P2B-2 收尾

“方框手势 → 直接截取手势包围范围 → 生成贴图”和“手势范围 → Vision OCR → 自动复制全文 → 截断摘要通知”已通过真机复验。贴图透明度、展开态复制、缩小态另存和多贴图管理也已完成复验。新版方框容错支持三边方框、大弧度转角和宽矩形，并分别输出顺时针和逆时针标识，已完成真机复验。

验收重点：拒绝屏幕录制权限时 P0/P1 与窗口动作完全不受影响；单屏及扩展屏的框选坐标正确；贴图不泄露捕获内容到网络。
