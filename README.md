# MouseTrails

MouseInc Windows 版概念的原生 macOS 手势工具，现以 MouseTrails 名称发布。当前已完成 P0、P1、P2-0，并交付 P2A/P2B 的首个可用切片：

- 菜单栏常驻应用
- 辅助功能权限检查
- 鼠标右键和触控板辅助点击共用同一套轨迹手势
- 手势轨迹浮层
- 八方向与组合方向识别
- 版本化 JSON 配置、旧配置迁移与按应用绑定
- 键盘快捷键动作
- schema 5 动作序列、自定义手势样本、旧配置迁移与完整性校验
- 辅助功能、屏幕录制与输入监控的独立权限状态、定向设置跳转和可拖拽应用授权助手
- SwiftUI 设置窗口，支持草稿编辑、实时校验与原子保存
- Accessibility 窗口居中、macOS 独立空间全屏切换、最小化和关闭动作
- 内置 S/W、顺逆时针方框和 8 种直角折线识别
- 自定义手势三次录制训练、缩略图预览、候选分差判定与冲突提示
- 方框模板与手势范围截图动作，可直接生成置顶贴图、复制或保存 PNG
- ScreenCaptureKit 按需授权，支持多显示器区域拼接和 macOS 13 单帧捕获兼容
- Vision 离线 OCR，识别手势包围范围后直接复制全文，并发送截断摘要通知
- 贴图支持拖动、折叠、悬停滚动调节透明度、展开态复制、缩小态另存与多贴图独立管理
- 设置中的轨迹预览、绑定排序、应用选择和就地冲突提示
- 配置导出、恢复前自动备份与独立权限状态展示
- GitHub Releases 更新检查，支持每日自动检查、手动检查、DMG 下载与 SHA-256 完整性校验

## 本地构建

项目使用 Swift Package Manager，并提供脚本生成可直接启动的 `.app`：

```sh
swift test
swift run MouseIncCoreCheck
./scripts/build-app.sh
open .build/MouseTrails.app
```

首次运行需要在“系统设置 → 隐私与安全性 → 辅助功能”中允许 MouseTrails。P0/P1 仍只需要辅助功能权限，不需要输入监控或屏幕录制。授权后返回菜单栏图标，确认状态显示为“监听状态：已启动”；若仍未启动，退出并重新打开应用。
权限页中的未授权项可直接点击：MouseTrails 会跳转到对应的系统设置，同时显示一个浮动授权助手，可将其中的应用图标拖入权限列表。

如果正在使用 Ice 等菜单栏管理工具，MouseTrails 可能被归入隐藏区。请在 Ice 中将菜单栏图标移入可见区；设置仅通过菜单栏图标进入。

## 鼠标与触控板操作

两种输入同时生效，应用不判断设备来源，也没有单独的模式开关。

- 鼠标：按住右键，拖动画出轨迹，然后松开。
- 触控板：做辅助点击（右键）并保持按下，拖动画出轨迹，然后松开。
- 只有短点击、未达到手势阈值时，仍会打开普通右键菜单。

“轻触来点按”形式的辅助右键会很快结束点击，不能维持按下状态，因此无法持续绘制轨迹。请使用可以持续按住的辅助点击方式；可在菜单栏中选择“打开触控板设置…”调整系统设置。

MouseTrails 不拦截双指滚动、捏合缩放、Mission Control 或系统的三/四指手势，这些操作仍完全由 macOS 处理。

## 更新与发布

应用从 [GitHub Releases](https://github.com/Nina-17/MouseTrails/releases) 查询最新正式版本。可在菜单栏或“设置 → 通用”中手动检查；自动检查默认开启，每 24 小时最多请求一次。发现新版本时会下载对应 DMG，校验 GitHub 记录的文件大小和 SHA-256 digest，然后交给 macOS 打开；应用不会在后台静默替换自身。

发布者在 `main` 上更新 `Resources/Info.plist` 版本并完成提交后，运行：

```sh
./scripts/tag-release.sh
```

脚本会推送 `v版本号` 标签，GitHub Actions 随后执行测试、构建 DMG、生成 SHA-256 文件并创建公开 Release。

配置文件位置：

```text
~/Library/Application Support/MouseIncMac/config.json
```

全新安装会从应用内置的 `default-config.json` 创建配置；覆盖安装和更新不会替换已有配置。目前内置默认值取自项目维护者的日常配置：绿色轨迹、边缘滚动开启以及 12 个手势绑定。

可从菜单栏选择“打开设置…”，编辑手势绑定及其动作序列。单段对角线手势标识为 `UP_LEFT`、`UP_RIGHT`、`DOWN_LEFT`、`DOWN_RIGHT`；8 种直角折线使用连字符，例如 `DOWN-RIGHT`（下→右）与 `UP-LEFT`（上→左）。
内置复杂模板为字母 `S`、`W` 与顺逆时针方框；模板不改变单方向与两段折线语义。

自定义手势：在“手势绑定”中新建或选中一个绑定，点击“录制自定义手势”，然后连续用右键画出同一轨迹 3 次。三次差异过大或轨迹过短时会要求重新录制；训练完成后设置动作并点击“保存设置”即可生效。固定手势已有绑定时依然优先执行。

全新安装的默认手势：

| 手势 | 动作 |
|---|---|
| 上 | Command+C |
| 下 | Command+V |
| 左 | Command+[ |
| 右 | Command+] |
| 下→右折线 | 关闭窗口或标签 |
| 顺时针方框 | 生成贴图 |
| 逆时针方框 | 离线 OCR 并复制结果 |
| 字母 S | 搜索选中文字 |
| 右上直线 | 切换全屏 |
| 左下直线 | 最小化窗口 |
| 下→左折线 | 关闭所有类似窗口 |
| 上→左折线 | 退出当前应用 |

完整范围与平台映射见 [移植规格](docs/PORTING_SPEC.md)；开发顺序、双轨边界与验收门槛见 [分阶段实现计划](docs/IMPLEMENTATION_PLAN.md)。

## 实现路线

- P0：工程/核心基线、可回归测试的手势核心、版本化配置与旧配置迁移。
- P1：触控板兼容，与鼠标共用右键轨迹链路，不干扰 macOS 原生手势。
- P2-0：双轨共享契约，包括动作序列、配置校验和独立权限状态。
- P2A：输入与效率功能，包括窗口、边缘、剪贴板和动作扩展。
- P2B：视觉与设置功能，包括设置 UI、截图、贴图和 OCR。P2A 与 P2B 在共享核心稳定后并行开发。

## 当前限制

- S/W 字母手势的进一步容错优化仍在暂挂清单。
- 手势目标暂以手势开始时的前台应用为准。
- 多显示器负坐标和不同缩放比例需要真机矩阵测试。
- 截图、贴图、离线 OCR、方框容错与简化边缘滚动均已通过真机验收。
- GitHub Actions 发布包当前为 ad-hoc 签名；要获得无 Gatekeeper 提示的公开分发体验，仍需 Developer ID Application 签名与 Apple 公证。
