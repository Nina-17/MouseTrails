# MouseTrails

MouseInc Windows 版概念的原生 macOS 手势工具，现以 MouseTrails 名称发布。当前开发版本为 **0.22.0**，P0、P1、P2A、P2B 已完成；本版本切换到 Sparkle 的签名验证与应用内安装更新。

- 菜单栏常驻应用
- 辅助功能与屏幕录制的独立权限检查、定向设置跳转和授权助手
- 鼠标右键和触控板辅助点击共用同一套轨迹手势
- 轨迹浮层、八方向直线、两段折线、顺逆时针方框与三次录制自定义手势
- 动作序列、按应用绑定、快捷键、窗口操作、系统视图与空间操作
- 左侧边缘滚动调节亮度、右侧边缘滚动调节音量
- SwiftUI 设置窗口：草稿编辑、实时校验、原子保存、轨迹预览、应用选择与配置导入导出
- 任务式交互教程：真实复制粘贴、浏览、搜索、窗口、贴图、OCR 与边缘滚动练习
- ScreenCaptureKit 区域截图、置顶贴图、Vision 离线 OCR、复制、保存与通知摘要
- Sparkle 更新检查、Ed25519 签名验证、下载和重启安装

## 本地构建

项目使用 Swift Package Manager，并提供脚本生成可直接启动的 `.app`：

```sh
swift test
swift run MouseIncCoreCheck
./scripts/build-app.sh
open .build/MouseTrails.app
```

本机迭代安装请使用：

```sh
./scripts/install-app.sh
```

该脚本要求可用的 Apple Development 签名，并固定覆盖 `/Applications/MouseTrails.app`；同时清理旧的 `~/Applications` 测试副本。这样可避免重复安装位置，以及每次代码变更后因 ad-hoc 签名身份变化而触发 macOS 隐私权限重授。

首次运行会自动打开任务式交互教程，并分别引导授予辅助功能和屏幕录制权限：辅助功能用于手势监听与动作执行，屏幕录制用于贴图、区域截图和 OCR。教程每次只展示当前任务，实际执行复制粘贴、搜索、隔离的窗口操作、贴图和 OCR，并在验证结果后自动推进；以后可从“设置 → 通用 → 使用教程”随时回看。辅助功能授权成功后会自动启动监听，无需退出应用。
权限页中的未授权项可直接点击：MouseTrails 会跳转到对应的系统设置，同时显示一个浮动授权助手，可将其中的应用图标拖入权限列表。

如果正在使用 Ice 等菜单栏管理工具，MouseTrails 可能被归入隐藏区。请在 Ice 中将菜单栏图标移入可见区；设置仅通过菜单栏图标进入。

## 鼠标与触控板操作

两种输入同时生效，应用不判断设备来源，也没有单独的模式开关。

- 鼠标：按住右键，拖动画出轨迹，然后松开。
- 触控板：做辅助点击（右键）并保持按下，拖动画出轨迹，然后松开。
- 只有短点击、未达到手势阈值时，仍会打开普通右键菜单。

“轻触来点按”形式的辅助右键会很快结束点击，不能维持按下状态，因此无法持续绘制轨迹。请使用可以持续按住的辅助点击方式；可在菜单栏中选择“打开触控板设置…”调整系统设置。

MouseTrails 不拦截双指滚动、捏合缩放、Mission Control 或系统的三/四指手势，这些操作仍完全由 macOS 处理。

## 更新与发布

应用通过 [Sparkle](https://sparkle-project.org/) 读取发布在 GitHub Pages 的更新清单。可在菜单栏或“设置 → 通用”中手动检查；自动检查默认开启，每 24 小时最多请求一次，自动下载可由用户单独开启。每个更新包都必须通过内置 Ed25519 公钥验证，Sparkle 随后会在退出并重新启动应用时替换安装副本，无需用户手动挂载 DMG 和拖拽应用。

发布者在 `main` 上更新 `Resources/Info.plist` 版本并完成提交后，运行：

```sh
./scripts/tag-release.sh
```

脚本会推送 `v版本号` 标签，GitHub Actions 随后执行测试、构建 DMG、生成 SHA-256 文件并创建公开 Release，同时生成并发布已签名的 Sparkle `appcast.xml`。发布工作流需要仓库 Secret `SPARKLE_ED25519_PRIVATE_KEY`；该私钥绝不能提交到仓库。

配置文件位置：

```text
~/Library/Application Support/MouseIncMac/config.json
```

全新安装会从应用内置的 `default-config.json` 创建配置；覆盖安装和更新不会替换已有配置。当前 schema 为 **7**，内置默认值为绿色轨迹、边缘滚动开启以及 10 个手势绑定；搜索手势会在教程中由用户录制并追加。“退出当前应用”以及系统视图/空间动作均可手动绑定，但不属于新安装默认配置。

可从菜单栏选择“打开设置…”，编辑手势绑定及其动作序列。单段对角线手势标识为 `UP_LEFT`、`UP_RIGHT`、`DOWN_LEFT`、`DOWN_RIGHT`；8 种直角折线使用连字符，例如 `DOWN-RIGHT`（下→右）与 `UP-LEFT`（上→左）。
内置复杂手势仅保留顺逆时针方框；字母手势可通过三次录制创建为个人自定义手势，教程会推荐录制 `S`。

自定义手势：在“手势绑定”中新建或选中一个绑定，点击“录制自定义手势”，然后连续用右键画出同一轨迹 3 次。三次差异过大或轨迹过短时会要求重新录制；训练完成后设置动作并点击“保存设置”即可生效。固定手势已有绑定时依然优先执行。

全新安装的默认手势：

| 手势 | 动作 |
|---|---|
| 上 | Command+C |
| 下 | Command+V |
| 左 | Command+[ |
| 右 | Command+] |
| 下→右折线 | 关闭窗口或标签 |
| 右上直线 | 切换全屏 |
| 左下直线 | 最小化窗口 |
| 下→左折线 | 关闭所有类似窗口 |
| 顺时针方框 | 生成贴图 |
| 逆时针方框 | 离线 OCR 并复制结果 |

教程会另行引导录制一个自定义手势（推荐 `S`）用于“搜索选中文字”；它会保存到用户自己的配置，而不是默认配置。

完整范围与平台映射见 [移植规格](docs/PORTING_SPEC.md)；开发顺序、双轨边界与验收门槛见 [分阶段实现计划](docs/IMPLEMENTATION_PLAN.md)。

## 实现路线

- P0：工程/核心基线、可回归测试的手势核心、版本化配置与旧配置迁移。
- P1：触控板兼容，与鼠标共用右键轨迹链路，不干扰 macOS 原生手势。
- P2-0：双轨共享契约，包括动作序列、配置校验和独立权限状态。
- P2A：输入与效率功能，包括手势、窗口、边缘滚动与动作扩展。
- P2B：视觉与设置功能，包括设置 UI、截图、贴图和 OCR。P2A 与 P2B 在共享核心稳定后并行开发。

## 当前限制

- 手势目标暂以手势开始时的前台应用为准。
- 多显示器负坐标和不同缩放比例仍需扩大真机矩阵测试。
- 截图、贴图、离线 OCR、方框容错和边缘滚动已通过真机验收；DockControl 空间切换已完成基础真机验证，动画手感仍可继续微调。
- GitHub Actions 发布包当前为 ad-hoc 签名；要获得无 Gatekeeper 提示的公开分发体验，仍需 Developer ID Application 签名与 Apple 公证。
