# MouseInc for macOS

MouseInc Windows 版的原生 macOS 功能重实现。当前已完成 P0、P1、P2-0，并交付 P2A/P2B 的首个可用切片：

- 菜单栏常驻应用
- 辅助功能权限检查
- 鼠标右键和触控板辅助点击共用同一套轨迹手势
- 手势轨迹浮层
- 八方向与组合方向识别
- 版本化 JSON 配置、旧配置迁移与按应用绑定
- 键盘快捷键动作
- schema 3 动作序列、延时、取消与配置校验
- 辅助功能、屏幕录制与输入监控的独立权限状态
- SwiftUI 设置窗口，支持草稿编辑、实时校验与原子保存
- Accessibility 窗口居中、最大化、还原、最小化和关闭动作
- 内置圆形及 C/M/Z 字母模板识别
- 设置中的轨迹预览、绑定排序、应用选择和就地冲突提示
- 配置导出、恢复前自动备份与独立权限状态展示

## 本地构建

项目使用 Swift Package Manager，并提供脚本生成可直接启动的 `.app`：

```sh
swift test
swift run MouseIncCoreCheck
./scripts/build-app.sh
open .build/MouseIncMac.app
```

首次运行需要在“系统设置 → 隐私与安全性 → 辅助功能”中允许 MouseIncMac。P0/P1 仍只需要辅助功能权限，不需要输入监控或屏幕录制。授权后返回菜单栏图标，确认状态显示为“监听状态：已启动”；若仍未启动，退出并重新打开应用。

## 鼠标与触控板操作

两种输入同时生效，应用不判断设备来源，也没有单独的模式开关。

- 鼠标：按住右键，拖动画出轨迹，然后松开。
- 触控板：做辅助点击（右键）并保持按下，拖动画出轨迹，然后松开。
- 只有短点击、未达到手势阈值时，仍会打开普通右键菜单。

“轻触来点按”形式的辅助右键会很快结束点击，不能维持按下状态，因此无法持续绘制轨迹。请使用可以持续按住的辅助点击方式；可在菜单栏中选择“打开触控板设置…”调整系统设置。

MouseIncMac 不拦截双指滚动、捏合缩放、Mission Control 或系统的三/四指手势，这些操作仍完全由 macOS 处理。

配置文件位置：

```text
~/Library/Application Support/MouseIncMac/config.json
```

可从菜单栏选择“打开设置…”，编辑手势绑定及其动作序列。单段对角线手势标识为 `UP_LEFT`、`UP_RIGHT`、`DOWN_LEFT`、`DOWN_RIGHT`；折线仍使用连字符，例如 `DOWN-RIGHT`。
内置复杂模板标识为 `CIRCLE`、`LETTER_C`、`LETTER_M`、`LETTER_Z`。模板只接管至少三段的复杂轨迹，不改变单方向与两段折线语义。

默认手势：

| 手势 | 动作 |
|---|---|
| 上 | Command+C |
| 下 | Command+V |
| 左 | Command+[ |
| 右 | Command+] |
| 下、右 | Command+W |

完整范围与平台映射见 [移植规格](docs/PORTING_SPEC.md)；开发顺序、双轨边界与验收门槛见 [分阶段实现计划](docs/IMPLEMENTATION_PLAN.md)。

## 实现路线

- P0：工程/核心基线、可回归测试的手势核心、版本化配置与旧配置迁移。
- P1：触控板兼容，与鼠标共用右键轨迹链路，不干扰 macOS 原生手势。
- P2-0：双轨共享契约，包括动作序列、配置校验和独立权限状态。
- P2A：输入与效率功能，包括窗口、边缘、剪贴板和动作扩展。
- P2B：视觉与设置功能，包括设置 UI、截图、贴图和 OCR。P2A 与 P2B 在共享核心稳定后并行开发。

## 当前限制

- 目前内置圆形和 C/M/Z，尚不包含 Windows 版全部字母与自定义训练。
- 手势目标暂以手势开始时的前台应用为准。
- 多显示器负坐标和不同缩放比例需要真机矩阵测试。
- 窗口置顶、Option 拖窗、边缘滚轮和热角待后续切片。
- 截图、OCR 和贴图属于后续 P2B 切片。
